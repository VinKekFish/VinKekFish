У ключей должны быть настройки:
	Минимальный размер открытого вектора инициализации (для режима обфускации)
	Шифр по-умолчанию, в том числе, шум и выравнивание - несколько разных настроек, если нужно
	Режим по-умолчанию (из нескольких)
	Сложность перебора по-умолчанию (минимальная)

Возможно, должен быть двойной режим шифрования: обфускация и, собственно, шифрование.
Ключ, в таком случае, можно быть просто двойным: для обфускации один, для чего-то другого - другой.

0. Режим "обфускации". Параллельный keccak, никаких настроек, спец. ключ с настройками, который используется только для обфускации.
Возможно, дополнительно, сильная обфускация: режим 4 и параллельный keccak

1. "Быстрый" многопоточный шифр.
	Предлагается keccak, каскадированный с TreeFish
		Происходит шифрование по 1 1024-хбитному блоку.
		Подготовка.
			Для всех блоков: подготавливается 5 keccak-состояний с помощью независимых ключей и констант.
			Для каждого блока подаётся клоны этих keccak без изменений
			Как рандомизировать номер блока?
			Каждый keccak получает на вход номер блока и хеширует его
			Два из них отдают ключ для TreeFish, два блок для TreeFish, один - tweak


2. Каскадный keccak (без threefish).
	
3. vinkekfish
	Набор для ключа 4096 битов с внутренним состоянием в 16384 битов

	Аналогично, 1/vinkekfish - параллельный шифр vinkekfish без treefish в конце

4. keccak, гамма или гамма с обратной связью по открытому тексту, без каскада.

5. Должна быть прослойка между разными keccak.
	Это может быть режим простой замены threefish, либо гаммирование по treefish
		Только гаммирование, только хардкор
	Как вариант, это может быть и простая обратная связь по шифротексту

6. Параллельный keccak

Для пользователя можно предоставить следующий выбор:
0. Обфускация паралелльным keccak и/или гаммой по threefish
1. Алгоритм 1.
2. Алгоритм 1 + гамма по 4
- 3. Алгоритм 1 + гамма по 4, 4 реверсивный с прослойкой 5 А есть ли смысл? Распараллеливание такого алгоритма не очень имеет смысл, лучше уж каскад сделать.
4. Алгоритм 1 + гамма по 4, наличие на выбор одного или двух вариантов из 2 и 3

Таким образом, предоставляется два базовых алгоритма: 1 и 4
+ возможность усиления любой комбинацией из 2 и 3

Теоретически, гамма также может быть расчитана vinkekfish аналогично алгоритму 1

Таким образом, получаем
Без выбора: Алгоритм 1

Первый выбор: любая комбинация (включая пустую) из
1. Простой параллельный keccak
2. Параллельный 1/vinkekfish

Второй выбор: любая комбинация (включая пустую) из
1. 4 реверсивный с прослойкой 5
2. 2 реверсивный с прослойкой 5
3. 3 реверсивный с прослойкой 5


Добавление шума
	Настройка добавления шума (один из вариантов)
	1. Нет шума
	2. Шум x2
	3. Шум x4
	4. Шум x8

Ручное выравнивание (один из вариантов)
	1. Стандартное выравнивание: граница, кратная степени 2-ки, 4-ки, 8-ки
	2. Выравнивание до какого-то числа значащих байтов (начиная от степени 6 - 64 байта) - до добавления шума

Перемешивание.
1. Без перемешивания
2. Слабое перемешивание (таблица номеров с xor, только для выравненных на степень двойки и более)
3. С перемешиванием (кратность перемешивания)

Что насчёт объединения файлов для скрытого шифрования?
Второй файл должен быть зашифрован так, чтобы быть не отличимым от шума.
	Это значит, что это тот же режим шифрования, т.к. больше настроек нет, их нигде нельзя сохранять
		При этом вектор инициализации может быть записан в шуме в другом канале или отдельно.
		Как вариант, тем же режимом шифрования может быть записан временный ключ, вектор инициализации, режим шифрования
	Либо используется спец. ключ с настройками по-умолчанию



Стандартные режимы "быстрой настройки" (потом можно поменять)
	Выбрать, обфускация или нет
	Далее
	0. Полностью настроить самому
	1. Большой файл слабый: 1 + простой параллельный keccak. Шум 0
	2. Большой файл сильный: 1 + параллельный keccak + Параллельный 1/vinkekfish. Шум 0
	3. Файл для флеш-памяти слабый: 1 + реверсивный 4 с прослойкой 5, гамма по 4. Шум x4, возможность скрытого файла, выравнивание не менее чем на 512 кб
	4. Файл для флеш-памяти сильный: 1 + реверсивные 2 и 3 (с прослойкой 5), гамма по 4. Шум x8, выравниванием не менее чем на 512 кб
	5. Минимальный размер: обфускация, шум x0, без выравнивания.
	6. Слабый: простой параллельный keccak и threefish, реверсивный 4, гамма по 4, Шум x2.
	7. Сильный: 1 + слабый + реверсивные 2 и 3, гамма по 4. Шум x8
	8. Базовый ключ: то же, что и сильный, но с более стойким преобразованием

	Можно и по-другому:
		Обфускация или нет
		Параллельный режим (для больших файлов) или нет
		Далее уже выравнивание для флеш-памяти
		Шум (всегда рекомендуется x8, для параллельных режимов выбора нет: 0)
